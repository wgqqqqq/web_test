# Agentic 工作流

Agentic 是 VCoder 的核心特性：描述目标，AI 自主规划并完成整个任务。

## 工作流程

```
用户 → 描述目标 → AI → [规划 → 执行 → 验证] → 结果
```

### 1. 意图理解

AI 会解析显式需求、推断隐式需求、识别约束条件：

```
输入: "添加用户登录功能"

AI 理解:
├─ 显式: 登录功能
├─ 隐式: 表单 UI、API 调用、状态管理、错误处理
└─ 约束: 使用现有组件库、遵循 API 模式、符合代码规范
```

### 2. 任务规划

自动分解为可执行步骤：

```
├─ 阶段 1: 分析现有代码，确定技术方案
├─ 阶段 2: 创建组件、API 服务、状态管理
├─ 阶段 3: 表单验证、错误处理、loading 状态
└─ 阶段 4: 测试验证
```

### 3. 自主执行

按计划执行：代码生成、文件操作、依赖安装、配置更新。

### 4. 持续验证

执行中持续检查：

```
[验证] LoginForm 组件
  ✓ TypeScript 类型正确
  ✓ 无 lint 错误
  ✓ 与项目风格一致
```

## 执行模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 自动 | 完全自主执行 | 简单明确任务 |
| 确认 | 关键步骤需确认 | 复杂任务（推荐） |
| 审查 | 逐一审查变更 | 敏感任务 |

## 中断控制

- **暂停/恢复** - 随时暂停，随时继续
- **取消** - 停止当前任务
- **回滚** - 撤销所有变更

## 最佳实践

**清晰描述**

```
❌ "改一下登录"
✅ "在登录表单添加'记住我'，用 localStorage 保存状态"
```

**提供上下文**

```
✅ "在 @src/components/Header.tsx 添加退出按钮"
```

**复杂任务分步**

```
1. "先创建数据模型和 API"
2. "实现用户界面"
3. "添加测试"
```
